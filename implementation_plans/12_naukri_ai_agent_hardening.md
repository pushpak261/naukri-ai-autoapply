# Implementation Plan - Naukri AI Agent Hardening

This plan addresses four critical logic bugs and logic gaps in the application orchestrator, form filler, screening question sorting, and GitHub Action workflow.

## User Review Required

> [!NOTE]
> None of the changes proposed modify the core logic/heuristics or change the user's config options. They are strictly hardening and correctness fixes to ensure stability, proper form-filling, and workflow persistence.

## Proposed Changes

### Component 1: Orchestration
Fix the concurrency race condition in the processing loop where producer and consumer fight over the same Playwright page context.

#### [MODIFY] [agent.py](file:///c:/Users/pushp/Music/AI_Agent_Naukri_refactored/refactored/src/orchestrator/agent.py)
Replace the producer-consumer task queue (`eval_queue` and `asyncio.gather(producer_task, consumer_task)`) with a single, synchronized sequential loop in `_process_jobs()`. This eliminates the race condition where `producer` navigates the browser page away while `consumer` is in the middle of filling out/submitting a job application.

---

### Component 2: Screening Question Sorting
Restore correct sorting order for screening questions when AI answers are generated.

#### [MODIFY] [question_answerer.py](file:///c:/Users/pushp/Music/AI_Agent_Naukri_refactored/refactored/src/ai/question_answerer.py)
In `answer_questions()`, map the original input indices from `ai_questions` back onto the parsed AI answers generated by `_ask_ai()`. This ensures they sort back to their original indexes correctly.

---

### Component 3: Form-Filling Capabilities
Add checkbox support to form-filling interactions.

#### [MODIFY] [apply.py](file:///c:/Users/pushp/Music/AI_Agent_Naukri_refactored/refactored/src/browser/apply.py)
Modify `_fill_answer()` to check if the `input_type` is `"checkbox"`. If so, click/toggle it based on the truthiness of the answer (e.g. if the answer is "yes", "true", "1"). This avoids raising Playwright exceptions when trying to type text into checkboxes.

---

### Component 4: Workflow Session Persistence
Add session sync step to the daily profile refresh workflow.

#### [MODIFY] [profile-refresh.yml](file:///c:/Users/pushp/Music/AI_Agent_Naukri_refactored/refactored/.github/workflows/profile-refresh.yml)
Append the `Encrypt and Commit Session` step (present in `auto-apply.yml`) to the bottom of the `profile-refresh.yml` workflow to ensure cookie refreshes are pushed back to the repository.
