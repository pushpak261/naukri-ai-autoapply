# SonarCloud CI Integration & Code Quality Assurance Plan

## Goal
Integrate continuous code quality inspection and test coverage tracking into the GitHub repository using SonarCloud and GitHub Actions, ensuring that no technical debt, bugs, or security vulnerabilities are introduced into the `naukri-ai-autoapply` agent codebase.

## Problem Context
As the project has grown with complex heuristic parsers, selenium browser automation, and AI prompt engineering, ensuring robust code quality is crucial. Previously, tests were run locally, and there was no automated Quality Gate blocking PRs if code quality dropped or if tests failed. 

## Proposed Architecture & Changes

### 1. SonarQube Configuration

#### [NEW] [sonar-project.properties](file:///c:/Users/pushp/Music/AI_Agent_Naukri_refactored/refactored/sonar-project.properties)
- Define `sonar.organization` as `pushpak261`.

### 2. GitHub Actions CI Pipeline

#### [NEW] [.github/workflows/sonarcloud.yml](file:///c:/Users/pushp/Music/AI_Agent_Naukri_refactored/refactored/.github/workflows/sonarcloud.yml)
- **Trigger**: Run the workflow on pushes to `main` and on any Pull Request (`opened`, `synchronize`, `reopened`).
- **Environment**: Use `ubuntu-latest` and `actions/setup-python@v4` (Python 3.12).
- **Dependencies**: 
  - Install standard pip dependencies.
  - Install dependencies from `requirements.txt` (core).
  - Install dependencies from `requirements-dev.txt` (testing utilities, crucially resolving `ModuleNotFoundError: No module named 'pytest_asyncio'`).
- **Testing & Coverage**: Run `pytest --cov=src --cov-report=xml` to execute all 108 unit tests and generate the `coverage.xml` payload.
- **Security & Execution**: 
  - Run `SonarSource/sonarcloud-github-action`.
  - **Crucial Security Fix**: Pin the GitHub action to an exact commit SHA (`@ba3875ecf642b2129de2b589510c81a8b53dbf4e`) instead of a branch like `@master` to pass SonarCloud's own `githubactions:S7637` security rule.
  - Expose `GITHUB_TOKEN` and `SONAR_TOKEN` secrets securely.

### 3. Codebase Bug Fixes (Found During Quality Assurance)

#### [MODIFY] [agent.py](file:///c:/Users/pushp/Music/AI_Agent_Naukri_refactored/refactored/src/naukri_agent/orchestrator/agent.py)
- **Bug**: The application was throwing an `AttributeError: 'ExclusionSettings' object has no attribute 'scam_threshold'` and subsequently failing to initialize the `ConsultancyScamSpecification` properly because it was being passed an invalid `threshold` keyword argument.
- **Fix**: Remove the `threshold=exclusions.scam_threshold` argument during instantiation. The `ConsultancyScamSpecification` v3 algorithm already defines an immutable class-level `SCAM_THRESHOLD = 80`.
